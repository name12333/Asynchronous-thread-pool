# Asynchronous-thread-pool
## 线程池设计
### 整体架构
1. 任务队列 (m_tasks)：存放等待执行的函数。
2. 工作线程 (worker)：一群“打工人”，不断从队列取任务并执行。
3. 管理者线程 (manager)：一个“监工”，负责根据任务负载动态增加或减少“打工人”的数量。
4. 异步结果 (future/packaged_task)：允许你在主线程提交任务后，在未来的某个时刻获取返回值。

## 线程池大小管理：
- 固定大小：
- 动态调整：根据任务负载动态调整线程池大小。 
## 任务队列管理：
- FIFO队列：按照任务提交顺序执行任务。
- 优先级队列：根据任务的优先级执行任务。
## 线程安全：
- 互斥锁
- 条件变量
- 原子变量

## 任务执行与错误处理：
- 任务执行过程中的异常需要适当处理，以避免线程池中的线程因未捕获异常而终止。
